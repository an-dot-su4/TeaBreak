# TeaBreak 設計書

「気分で選ぶ紅茶レコメンドアプリ」の設計。要件定義書（`docs/要件定義書.md`）に基づく。

- **文書バージョン**：1.0
- **作成日**：2026-07-05
- **関連 Issue**：#1

---

## 1. アーキテクチャ概要

- **形態**：ビルドレスの静的 PWA（素の HTML/CSS/JS、フレームワーク・バンドラ不使用）。
- **実行**：すべてブラウザ内で完結。外部 API 通信なし。
- **配信**：GitHub Pages（`app/` を公開ルート）。
- **状態**：画面内の JS 変数のみ（永続化なし）。

```
[ユーザー] ──選択──▶ app.js（状態機械）
                        │  参照
                        ▼
                     data.js（質問・茶葉・ブランド・リンク生成）
                        │
                        ▼
                  結果カード描画（index.html 内のDOM）
sw.js ＋ manifest：オフライン & インストール
```

## 3. データモデル（`data.js`）

### 3.1 茶葉カタログ `TEAS`
```js
{
  id: "darjeeling-2nd",
  name: "ダージリン セカンドフラッシュ",
  origin: "インド",
  profile: "マスカットのような香り。クリアで華やか。",
  caffeine: "medium",          // 'high' | 'medium' | 'low' | 'none'
  hot: true,                    // ホットに向くか
  iced: true,                   // アイスに向くか
  milk: "no",                   // 'good' | 'ok' | 'no'（ミルク適性）
  flavor: "plain",              // 'plain' | 'flavored' | 'spiced'
  scenes: ["focus", "special"], // 対応シーンID
  brew: "ストレートで香りを楽しむ。",
  brandIds: ["lupicia", "mariage", "twinings"]
}
```

**掲載する茶葉（初期）**：ダージリン1st/2nd、アッサム、ニルギリ、ウバ、ディンブラ、ヌワラエリヤ、キーモン、雲南、ケニアCTC、アールグレイ、イングリッシュブレックファスト、マサラチャイ、デカフェ紅茶、ルイボスティー、カモミール。

### 3.2 ブランド `BRANDS`
```js
{
  id: "lupicia",
  name: "ルピシア",
  country: "日本",
  officialSearch: "https://www.lupicia.com/",   // 公式（トップ or 検索）。無ければ null
}
```
**保持するブランド（初期）**：ルピシア／日東紅茶／トワイニング／フォートナム&メイソン／マリアージュフレール／クスミティー／ロンネフェルト／ディルマ／ワグバクリ／ケニア系（無指定はブランド非依存の一般検索）。

### 3.3 シーン定義 `SCENES`
`focus / relax / morning / afterMeal / refresh / warm / bedtime / special` の8種。各 `id, label, emoji, accentVar(CSS変数名)`。

### 3.4 質問フロー `QUESTIONS`
```js
[
  { id: "scene",    text: "今の気分は？",         options:[{label,value,emoji}], },
  { id: "temp",     text: "ホット？アイス？",      options:[hot|iced], },
  { id: "milk",     text: "ミルクは入れる？",      options:[yes|no|either], },
  { id: "caffeine", text: "カフェインは控えたい？", options:[yes|no], defaultFrom:"scene==bedtime" }
]
```

## 4. レコメンドアルゴリズム（`app.js`）

入力：回答 `{ scene, temp, milk, caffeine }`。

1. **ハードフィルタ（除外）**
   - `caffeine === 'avoid'` → `tea.caffeine === 'none'` のみ残す。
   - `temp === 'iced'` → `tea.iced === true` のみ。
   - `temp === 'hot'` → `tea.hot === true` のみ。
2. **スコアリング（加点）**
   - シーン一致：`tea.scenes.includes(scene)` → +5
   - ミルク：`milk==='yes'` かつ `tea.milk==='good'` → +3、`milk==='no'` かつ `tea.milk==='no'` → +2、`either` → +1
   - 温度：`temp` に合致（hot/iced）→ +1
   - 就寝前×none：`scene==='bedtime'` かつ `caffeine==='none'` → +2
3. **並べ替え**：スコア降順、同点は `caffeine` の適合や `id` 安定ソート。
4. **上位 N=最大3件**を採用。0件の場合はフィルタを緩めて（温度条件のみ維持）フォールバック提示＋メッセージ。

> アルゴリズムは純粋関数 `recommend(answers, TEAS): Tea[]` として実装し、テスト容易にする。

## 5. 購入リンク生成（純粋関数）

```js
function buildLinks(brandName, teaName, brand) {
  const q = encodeURIComponent(`${brandName} ${teaName}`.trim());
  return {
    amazon: `https://www.amazon.co.jp/s?k=${q}`,
    rakuten: `https://search.rakuten.co.jp/search/mall/${q}/`,
    official: brand?.officialSearch
      ? brand.officialSearch
      : `https://www.google.com/search?q=${q}%20公式`
  };
}
```
- ブランド非依存の一般提案では `brandName` を空にし、茶葉名のみで検索。
- すべて `rel="noopener noreferrer"`、`target="_blank"`。

## 6. 画面・UI 設計

### 6.1 レイアウト
- 縦1カラムのチャット画面。上部ヘッダー（アプリ名・テーマは自動）、中央にチャットログ、下部に選択肢チップ（クイックリプライ）。
- 結果は「結果カード」をチャットログ内に差し込む。

### 6.2 コンポーネント
| 部品 | 説明 |
|------|------|
| メッセージ吹き出し（bot/user） | 質問は bot 側、選んだ回答は user 側に表示。 |
| クイックリプライ・チップ | 選択肢ボタン群。選ぶと user 吹き出し＋次の質問へ。 |
| 結果カード | 茶葉名・理由・淹れ方・ブランド別リンク（Amazon/楽天/公式）。 |
| 注意バナー | 感覚的提案・カフェイン・アフィリエイト無しの注記。 |
| 「もう一度」ボタン | フロー初期化。 |

### 6.3 デザイントークン（`styles.css`）
- 既存資料の紅茶パレットを踏襲：暖色クリーム地／深い焙じ茶ダーク地、アクセント＝銅色 `#A8571E`。
- シーン別アクセント（集中=緑系、リラックス=ラベンダー、朝=琥珀…）を結果や吹き出しに反映。
- ライト/ダークは CSS カスタムプロパティ＋ `prefers-color-scheme`。
- フォント：見出し＝セリフ/明朝、本文＝システムサンス。

### 6.4 アクセシビリティ
- チップは `<button>`。フォーカスリング可視。`aria-live="polite"` でチャットログ更新を通知。
- コントラスト比 AA を満たす配色。

## 7. PWA 設計

### 7.1 `manifest.webmanifest`
- `name`, `short_name`="TeaBreak", `start_url`="./", `scope`="./", `display`="standalone",
  `theme_color`/`background_color`＝紅茶パレット、`icons`（192/512/maskable）。

### 7.2 `sw.js`
- `install`：アプリシェル（index.html, styles.css, app.js, data.js, manifest, icons）をプリキャッシュ。
- `activate`：旧キャッシュ削除。
- `fetch`：cache-first（外部の Amazon/楽天等へのナビゲーションはキャッシュせず素通し）。
- キャッシュ名にバージョンを持たせ、更新時に切替。

### 7.3 アイコン
- ティーカップをモチーフにした簡易アイコンを生成（背景＝クリーム/銅色）。192・512・maskable(512, セーフエリア考慮)。

## 9. テスト計画

### 9.1 ロジック（純粋関数）
- `recommend()`：各シーンで1件以上返る／就寝前×控えたい→none のみ／アイス→iced のみ。
- `buildLinks()`：URL 形式とクエリ内容（「ブランド 茶葉」）。

### 9.2 E2E（Chromium/Playwright）
- 会話フローを最後まで進めて結果カード表示を確認。
- 生成リンクの `href` を検証。
- ライト/ダーク、スマホ幅（375px）/PC幅で崩れないことをスクリーンショット確認。
- `sw.js`・`manifest` 読み込みとコンソールエラー無しを確認。
