# TeaBreak 要件定義書

「今の気分」を選ぶと、おすすめの紅茶とブランド・購入リンクを提案する Web アプリ（PWA）。

- **文書バージョン**：1.0
- **作成日**：2026-07-05
- **対象リポジトリ**：`an-dot-su4/teabreak`
- **関連 Issue**：#1

---

## 1. 背景と目的

### 背景
別リポジトリ `an-dot-su4/claude-library` に、紅茶の調査資料（産地・茶葉・グレード・国別ブランド・消費量）と、8シーンの「気分別おすすめ紅茶」資料が整備済み。これらは読み物としては充実しているが、**「今の自分に合う一杯」を選ぶ体験**にはなっていない。

### 目的
資料の知識を、**気分から紅茶を提案してくれるインタラクティブなアプリ**に落とし込む。さらに「どこで買えるか」まで案内し、飲みたい気持ちから購入までをつなぐ。

### ゴール（このプロダクトが提供する価値）
- 迷わず「今の気分に合う紅茶」に出会える。
- 茶葉だけでなく、**具体的なブランド**と**購入先（複数サイト）**まで分かる。
- スマホ（Android）でも PC でも同じ体験。インストールしてすぐ使える。

## 2. 想定ユーザーと利用シーン

| 種別 | 想定 |
|------|------|
| 主なユーザー | 紅茶を飲みたいが銘柄選びに迷う人。初心者〜中級者。 |
| 利用デバイス | Android スマートフォン、PC（ブラウザ）。 |
| 利用シーン | 「集中したい」「リラックスしたい」「就寝前」など、その時の気分で一杯を選びたいとき。買い足したいとき。 |
| 前提知識 | 不要（専門用語は最小限＋補足）。 |

## 3. 機能要件

### FR-1 チャット風の気分ヒアリング
- 選択肢ボタン（クイックリプライ）で会話するように進む。
- 質問は5問：①気分・シーン（8種）②ホット/アイス ③ミルクの有無 ④フレーバー（香り付き）の好み ⑤カフェインを控えるか。
- 直前の回答はチャット履歴として画面に残る。

### FR-1b フレーバーの好み
- 「おまかせ／香り付きがいい／茶葉そのもの（プレーン）」を選べる。
- 「香り付きがいい」を選ぶと、アールグレイ等の**フレーバーティー**が優先提案される。

### FR-2 レコメンド（提案ロジック）
- 回答をもとに茶葉を**スコアリングし、上位1〜3件**を提案する。
- ハード条件（例：「カフェインを控えたい」→ノンカフェインのみ、「アイス」→アイス可の茶葉のみ）で除外。
- シーン一致・温度・ミルク嗜好で加点。

### FR-3 結果表示
- 各提案に「茶葉名／おすすめ理由（感覚的な一言）／淹れ方のヒント」を表示。
- 「なぜこれ？」が伝わる短い説明を付ける。

### FR-4 ブランドと購入リンク
- 各茶葉に合う**ブランドを1〜3件**提示。
- ブランドごとに購入先リンクを **3サイト**表示：**Amazon.co.jp／楽天市場／ブランド公式ストア**。
- リンクは「ブランド名＋茶葉名」の**検索リンク**を基本とする（商品直リンクはリンク切れリスクが高いため）。公式ストアURLが分かるブランドは公式トップ/検索へ。
- リンクは新規タブで開く（`target="_blank"` ＋ `rel="noopener"`）。

### FR-5 やり直し・再提案
- 「もう一度」で最初の質問に戻れる。
- （任意）「別の候補を見る」で次点を表示。

### FR-6 注意書きの明示
- 「感覚的な提案であり感じ方には個人差がある」旨を明示。
- カフェインに関する注意（就寝前はデカフェ／ノンカフェイン推奨）を明示。
- 購入リンクは検索リンクで在庫・価格は各サイト依存、**アフィリエイト無し**であることを明示。

### FR-7 PWA（インストール）
- Android のホーム画面に追加でき、アプリのように起動できる。
- オフラインでも起動・利用できる（静的アセットをキャッシュ）。

## 4. 非機能要件

| 分類 | 要件 |
|------|------|
| 対応環境 | 最新の Chrome/Edge/Safari（PC・Android）。 |
| レスポンシブ | スマホ幅〜PC幅で崩れない。横スクロールを起こさない。 |
| テーマ | ライト／ダーク両対応（OS 設定に追従）。 |
| パフォーマンス | 追加の外部ネットワーク通信なしで即応答（ローカル完結）。初期表示は軽量。 |
| オフライン | Service Worker により2回目以降はオフラインでも動作。 |
| アクセシビリティ | キーボード操作可、フォーカス可視、十分なコントラスト、ボタンに適切なラベル。 |
| 言語 | 日本語のみ。 |
| プライバシー | 個人情報を収集しない。回答データは端末外に送信しない。 |

## 5. 制約・前提

- **バックエンド無し**（サーバー不要の静的サイト）。
- **LLM 不使用**（API キー・従量課金・外部AIサービスに依存しない）。
- **ホスティング**：GitHub Pages（`app/` を公開）。
- **ビルドツール不要**（素の HTML/CSS/JS。Node/npm ビルド前提にしない）。
- データ（茶葉・ブランド・リンク）はアプリ内に静的に保持し、容易に追記・修正できる形にする。

---

## 6. スコープ外（今回作らないもの）

- 自由入力の本格 AI チャット（将来 Claude API ＋簡易サーバーで拡張余地を残す）。
- ユーザー登録・履歴・お気に入り保存。
- 価格取得・在庫連携・アフィリエイト。
- Play ストア配信（ネイティブ化）。多言語対応。

---

## 7. 成功基準（受け入れ条件）

- [ ] 8シーンいずれを選んでも、最後まで進むと**1件以上の茶葉が提案**される。
- [ ] 各提案に**ブランド**と**3サイトの購入リンク**が表示され、リンクが正しく開く（検索クエリに「ブランド 茶葉」が入る）。
- [ ] 「就寝前」＋「カフェインを控えたい」で、**ノンカフェイン系のみ**が提案される。
- [ ] 「香り付きがいい」を選ぶと、**フレーバーティー**が提案に含まれる。
- [ ] 「アイス」を選ぶと、アイス不適の候補が除外される。
- [ ] ライト／ダーク、スマホ幅／PC幅で表示が崩れない。
- [ ] オフライン（2回目以降）でも起動・操作できる。
- [ ] 感覚的提案・カフェイン注意・アフィリエイト無しの注意書きが表示される。
