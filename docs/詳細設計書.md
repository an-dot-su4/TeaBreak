## 1. 技術方針

- 素の HTML/CSS/JS（フレームワーク・バンドラ不使用）。`<script>` は非モジュールで読み込み。
- `data.js` が `window.TEABREAK_DATA` を公開し、`app.js` が参照（ビルド不要、file:// でも動作）。
- 状態は `app.js` 内のオブジェクトのみ（永続化なし）。

## 2. ファイル構成

```
app/
  index.html            # 器（ヘッダー / チャットログ / 選択肢 / 注意書き）

  data.js               # SCENES / QUESTIONS / TEAS / BRANDS / linkビルダー
```

## 3. データスキーマ（`data.js`）

### 3.1 SCENES
```js
{ id:"focus", label:"集中したい", emoji:"🎯", accent:"--s-focus" }
// focus, relax, morning, afterMeal, refresh, warm, bedtime, special の8種
```

### 3.2 QUESTIONS（5問）
```js
[
  { id:"scene",    text:"今の気分は？",           type:"scene" },  // SCENESから生成
  { id:"temp",     text:"ホット？アイス？",        options:[
      {label:"ホット",value:"hot",emoji:"♨️"},{label:"アイス",value:"iced",emoji:"🧊"}] },
  { id:"milk",     text:"ミルクは入れる？",         options:[
      {label:"入れる",value:"yes"},{label:"入れない",value:"no"},{label:"どちらでも",value:"either"}] },
  { id:"flavor",   text:"フレーバー（香り付き）は？", options:[
      {label:"おまかせ",value:"any"},{label:"香り付きがいい",value:"flavored"},{label:"茶葉そのもの",value:"plain"}] },
  { id:"caffeine", text:"カフェインは控えたい？",     options:[
      {label:"控えたい",value:"avoid"},{label:"気にしない",value:"ok"}],
      defaultWhen:{ scene:"bedtime", value:"avoid" } }
]
```

### 3.3 TEAS
```js
{
  id:"darjeeling-2nd", name:"ダージリン セカンドフラッシュ", origin:"インド",
  profile:"マスカットのような香り。クリアで華やか。",
  caffeine:"medium",           // 'high'|'medium'|'low'|'none'
  hot:true, iced:true,
  milk:"no",                   // 'good'|'ok'|'no'
  flavor:"plain",              // 'plain'|'flavored'|'spiced'
  scenes:["focus","special"],
  brew:"ストレートで香りを楽しむ。",
  brandIds:["lupicia","mariage","twinings"]
}
```
**掲載（約22種）**：紅茶（darjeeling-1st/2nd, assam, nilgiri, uva, dimbula, nuwara, keemun, yunnan, kenya-ctc, english-breakfast）／フレーバード（earl-grey, caramel, fruit, marco-polo, lapsang）／スパイス（masala-chai）／**カフェインレス・ハーブ**（decaf, rooibos, chamomile, hibiscus, peppermint, honeybush）。カフェインレスはアイス可・香り付き/プレーンを揃え、「控えたい」選択時の候補を厚くしている。

## 4. レコメンドアルゴリズム

```
function recommend(answers, TEAS):
  cands = TEAS.filter(t =>
      (answers.caffeine!=='avoid' || t.caffeine==='none') &&
      (answers.temp!=='iced'      || t.iced) &&
      (answers.temp!=='hot'       || t.hot))
  for t in cands: t._score =
      (t.scenes.includes(answers.scene) ? 5 : 0)
    + milkScore(answers.milk, t.milk)          // yes: good+4/ok+2/no-3, no: no+3/ok+1/good-3, either:0（ミルク希望を尊重）
    + (tempMatch ? 1 : 0)
    + flavorScore(answers.flavor, t.flavor)     // flavored&(flavored|spiced):+3 / plain&plain:+2,plain&flavored:-3 / any:0
    + ((answers.scene==='bedtime'&&t.caffeine==='none') ? 2 : 0)
  sort by _score desc, then stable by id
  positive = scored with _score>0
  if positive not empty:
    // B: 上位と乖離した“弱い候補”は出さない（無理に3件にしない）
    floor = max(2, positive[0]._score - 6)
    result = positive with _score>=floor, take up to 3（0件なら最上位1件）
  else:
    // ハード条件（カフェイン/温度）を維持したままスコア順で採用（フォールバック）
    result = top 3 of cands
  return result
```
- `recommend` は副作用なしの純粋関数（テスト対象）。

## 5. 購入リンク生成

```js
function buildLinks(brandName, teaName, brand){
  const q = encodeURIComponent(`${brandName} ${teaName}`.trim());
  return {
    amazon:  `https://www.amazon.co.jp/s?k=${q}`,
    rakuten: `https://search.rakuten.co.jp/search/mall/${q}/`,
    official: brand && brand.officialSearch
      ? brand.officialSearch
      : `https://www.google.com/search?q=${q}%20公式`
  };
}
```
- リンクは `target="_blank" rel="noopener noreferrer"`。

## 7. デザイントークン（`styles.css`）

- パレット（紅茶）：`--ground/--surface/--ink/--muted/--accent(#A8571E)/--border`。ダークは焙じ茶系に再定義。
- シーン別アクセント `--s-focus`… を結果カードの上帯・チップ選択に使用。
- `prefers-color-scheme` ＋ `:root[data-theme]` の両対応。
- フォント：見出し=セリフ/明朝、本文=システムサンス。数値は `tabular-nums`。

## 8. PWA

### 8.1 manifest.webmanifest
`name:"TeaBreak 気分で選ぶ紅茶", short_name:"TeaBreak", start_url:"./", scope:"./", display:"standalone", background_color/theme_color=紅茶パレット, icons:[192,512,maskable]`。

### 8.2 sw.js
- `CACHE="teabreak-v2"`。`install` でアプリシェルをプリキャッシュし `skipWaiting`。
- `activate` で旧キャッシュ削除＋`clients.claim`。
- `fetch`：同一オリジンの GET は**ネットワーク優先**（成功時はキャッシュ更新／失敗時はキャッシュへフォールバック）。更新が確実に反映されるようにするため。外部（Amazon等）は素通し。

### 8.3 icons
ティーカップを描いた簡易アイコンを生成（背景=クリーム、絵柄=銅色）。192 / 512 / maskable(512, セーフエリア内)。

## 9. デプロイ（`.github/workflows/pages.yml`）

- トリガ：`main` への push（＋手動 `workflow_dispatch`）。
- ジョブ：`actions/configure-pages` →`actions/upload-pages-artifact`（`path: app`）→`actions/deploy-pages`。`permissions: pages:write, id-token:write`。
- **一度だけ手動**：Settings→Pages→Source=「GitHub Actions」。
- URL：`https://an-dot-su4.github.io/teabreak/`。

## 10. テスト計画

### 10.1 ロジック
- `recommend()`：全8シーンで1件以上／就寝前×avoid→none のみ／iced→iced のみ／flavor=flavored→フレーバー含む。
- `buildLinks()`：3 URL の形式とクエリ内容。

### 10.2 E2E（Chromium/Playwright, `executablePath:/opt/pw-browsers/chromium`）
- 5問を進めて結果カード表示。
- 生成リンク `href` 検証。
- ライト/ダーク・375px/PC幅のスクリーンショット。
- `manifest`/`sw` 読み込み・コンソールエラー無し。
