## 1. システム概要

TeaBreak は、ユーザーが「今の気分」をチャット風に選ぶと、**おすすめの紅茶・合うブランド・購入リンク**を提案する Web アプリ。知識資料（claude-library の紅茶ガイド）を「選ぶ体験」に変換する。

```
気分を選ぶ  ─▶  紅茶を提案  ─▶  ブランドと購入先を案内
（チャット風）    （理由・淹れ方）    （Amazon/楽天/公式）
```

## 2. 設計思想（なぜこの構成か）

| 判断 | 理由 |
|------|------|
| **静的 PWA（サーバー無し）** | 個人開発で運用負荷ゼロ・無料公開・高速。個人情報を扱わないため DB 不要。GitHub Pages に載せられる。 |
| **チャット“風”UI（スクリプト型・LLM不使用）** | 会話体験の親しみやすさを保ちつつ、API 課金・鍵管理・サーバーを不要にする。応答は確定的で外れない。将来 AI 化する余地は残す。 |
| **データ（茶葉・ブランド）を1ファイルに集約** | 非エンジニアでも追記・修正しやすく、資料（claude-library）との対応を保ちやすい。 |
| **購入リンクは検索リンク** | 商品直リンクはリンク切れ・在庫切れが起きやすい。検索リンクなら常に開けて、ユーザーが選べる。複数サイトで購入環境の差を吸収。 |

## 3. 機能一覧（外部から見た機能）

| ID | 機能 | 概要 |
|----|------|------|
| F1 | 気分ヒアリング | チャット風に5問を選択式で質問。 |
| F2 | レコメンド | 回答から紅茶を1〜3件提案。 |
| F3 | 提案の説明 | 各紅茶に理由・淹れ方を表示。 |
| F4 | ブランド＆購入導線 | 紅茶ごとにブランドと Amazon/楽天/公式リンク。 |
| F5 | やり直し | 最初から選び直す。 |
| F6 | 注意表示 | 感覚的提案・カフェイン・アフィリエイト無しの明示。 |
| F7 | PWA | インストール・オフライン利用。 |

## 4. 画面構成の概念

- **単一画面**のチャット。上から「ヘッダー」「チャットログ（質問と回答と結果カードが積み上がる）」「選択肢（クイックリプライ）」。
- 提案は会話の続きとして**結果カード**で差し込む。読み進める延長で購入導線まで到達する。

```
┌───────────── TeaBreak ─────────────┐
│ [bot] 今の気分は？                   │
│                 [user] 集中したい     │
│ [bot] ホット？アイス？ …（5問）        │
│ ┌── 結果カード ─────────────┐         │
│ │ ダージリン2nd / 理由 / 淹れ方 │         │
│ │ ルピシア  [Amazon][楽天][公式] │        │
│ └──────────────────────────┘         │
│ [ 選択肢チップ … ]  [もう一度]         │
└────────────────────────────────────┘
```

## 5. データの概念モデル

3つの概念とその関係：

```
シーン(Scene) ──推奨──▶ 紅茶(Tea) ──合う──▶ ブランド(Brand)
   8種            属性:カフェイン/温度/           公式ストア/
（集中/就寝前…）    ミルク/フレーバー              購入検索リンク
```

- **シーン**：気分の入口（8種）。紅茶が「どのシーン向きか」を持つ。
- **紅茶**：提案の主役。カフェイン量・ホット/アイス適性・ミルク適性・フレーバー種別・対応シーンを属性に持つ。
- **ブランド**：紅茶を実際に買うための銘柄。購入リンクを生成する起点。

## 6. 会話フローの概念（5問）

1. **気分・シーン**（8種）… 提案の主軸
2. **ホット / アイス** … 温度で候補を絞る
3. **ミルクの有無** … ミルク適性で重み付け
4. **フレーバーの好み**（おまかせ / 香り付き / プレーン）… フレーバーティーの出し分け
5. **カフェインを控えるか**（就寝前は既定でON）… 夜間の安全側

→ 回答から紅茶を点数化し、上位1〜3件を提案。詳細は詳細設計書。

## 7. 外部連携

- **購入サイト**：Amazon.co.jp／楽天市場／ブランド公式（またはWeb検索）。**ページ遷移のみ**で、データ連携・APIは使わない。新規タブで開く。
- それ以外の外部通信は行わない（プライバシー・オフライン性を担保）。

## 8. 非機能方針

- **対応**：PC・Android の最新ブラウザ。レスポンシブ。ライト/ダーク自動。
- **性能**：ローカル完結で即応答。アプリシェルは軽量。
- **オフライン**：Service Worker により2回目以降も起動可能。
- **アクセシビリティ**：キーボード操作・フォーカス可視・十分なコントラスト。
- **プライバシー**：収集・送信なし。

## 9. 制約・前提

- バックエンド無し／LLM 無し／ビルドツール前提にしない（素の HTML/CSS/JS）。
- ホスティングは GitHub Pages。
- 提案は**感覚的**なものであり、カフェイン感受性等には個人差がある旨を明示。

## 10. 将来拡張（概念）

- 「入力 → 意図抽出 → 既存の提案ロジック」の形で、後から **AI 自由入力チャット**（Claude API）を差し込める構造にする。
- 茶葉・ブランドはデータ追記のみで拡張。
